The phases, in order

phase 0 · next
Foundations — tokens, component kit, Next.js shell
Port tokens.css to :root + [data-theme=dark], self-host the three fonts, build the atoms (Eyebrow … GateBadge … ThemeToggle) and chrome (NLNav / NLFooter), wire light/dark end to end. Accept: a blank themed page renders in both themes with no console errors; a kitchen-sink route shows every component; tokens match design-system.md exactly.
phase 1
The Home (V1) — the approved front door
Build sections 1–8 of surfaces.md, the three-persona personalization demo with the sg-fade re-cut, and the responsive collapses. Accept: pixel-matches Newsletters - Home.html in both themes; renders without JS.
phase 3
The four surfaces, from Surface objects
Recreate Newsletter / Article / Report / Show in the real stack, each rendering from a typed Surface with a real GateBadge — our Rev1 renderer is the seed. Accept: matches each reference; gate is real, not hardcoded; the Article prints clean to PDF.
phase 4
The publish loop — human in the loop
Ingest adapters → the agentic distill() behind one swappable boundary → open the draft as a real PR against /content → merge renders to the Library and sends per-reader. Accept: a fixture event flows end to end; no auto-publish path exists.
phase 5
Library, private corpora & MCP
The Hub archive; encrypted-at-rest corpora that never transmit; one MCP server per source system so data and corpora stay in the operator's environment. Accept: a corpus round-trips locally encrypted; the distiller reads sources only through MCP.
phase 6
Open-source release
Slot-marked templating for operator repopulation, MIT + self-host docs, verified no-telemetry / no-JS / WCAG AA. Accept: a fresh operator clones, repopulates slots, self-hosts, and publishes their first surface from the README alone.
