<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| config path string → filesystem | An operator-supplied path could escape the repo root (path traversal) |
| config YAML content → minted Claims | Untrusted config text becomes Claim/KpiItem values consumed downstream |
| parsed values → Trace offsets | A mis-computed offset would mis-attribute provenance |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | path resolution in the load function | mitigate | Mirror `worksurface.capture_files`: `resolved.relative_to(root)` raises ValueError on any path escaping root; missing → FileNotFoundError, non-utf8 → UnicodeDecodeError (fail loud, never silent). |
| T-02-02 | Information disclosure / Tampering | config content minted into Claims | accept | Values are stored as verbatim plain-string data (`span`), never executed or evaluated; faithful extraction never interprets content. Downstream HTML rendering (existing/Phase 2 renderer) owns escaping — out of scope here. No private-corpus data enters a Source. |
| T-02-03 | Tampering / Repudiation | trace minting | mitigate | `Trace.from_source` is the SOLE minting path; it validates `0<=start<=end<=len` before slicing (raises on a bad range) and pins `content_hash`. No hand-minted hash, no fabricated offset — provenance cannot silently lie. |
| T-02-04 | Elevation of Privilege | YAML deserialization | mitigate | Parsing is delegated to Plan 01-01's `load_config`, which uses `yaml.safe_load` only — no arbitrary object construction. |
| T-02-SC | Tampering (supply chain) | PyYAML | mitigate | No new package added here; inherits Plan 01-01's version-pinned, verified, lazy-boundary PyYAML. |
</threat_model>
