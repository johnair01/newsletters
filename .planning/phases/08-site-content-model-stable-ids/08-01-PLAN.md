<objective>
Build the deterministic, AI-free identity core for the Library: a new `src/newsletters/site.py`
holding a stdlib `slugify()`, an append-only ID `Ledger`, and the typed `Site / Collection / Page`
content model with `Site.from_surfaces()` and a `Site.by_slug()` cross-link resolver. Prove the
stability invariant (ROADMAP success criterion 2) with a TDD test suite, and seed the committed
ledger at `content/rev1/ids.json`.

This is the data layer of Phase 8 (SITE-01). It moves identity OUT of the presentation tier and
INTO a deterministic core model. Identity is a pure function of content + the append-only ledger:
same input → same ledger → same IDs. List position is NEVER the source of identity.

Purpose: stable per-surface IDs (criterion 1) that survive reorder/insert (criterion 2), so refs
and cross-links stop rotting. Implements locked decisions L1 (ID conventions), L4 (stdlib slugify),
L5 (append-only ledger), L6 (by_slug resolver only — no kanban/nav/Home), L7 (stability test).
Output is consumed by Plan 02 (renderer wiring + spec update).

Output: `src/newsletters/site.py`, `tests/test_site.py`, seeded `content/rev1/ids.json`, exports in
`src/newsletters/__init__.py`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| author title → slug → filename | A surface title flows through slugify() into `f"{slug}.html"`; a malformed title must not produce a path-traversal filename. |
| committed ledger file → model | `content/rev1/ids.json` is human-reviewed committed config, not user input; loaded via json.loads (no eval). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering | slugify() → `f"{slug}.html"` | mitigate | slugify strips everything outside [a-z0-9-]; test_slug_is_filename_safe asserts no "/", ".", "\\", or whitespace ever appears in output (ASVS V5). |
| T-08-02 | Tampering | Ledger.load(ids.json) | accept | Ledger is a committed, human-reviewed file, not external input; loaded via json.loads (never eval). Low risk. |
| T-08-03 | Tampering | non-deterministic ledger writes | mitigate | save() uses json.dumps(sort_keys=True) so a no-op rebuild is byte-stable; test_ledger_json_is_stable guards it. |
| T-08-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this phase — pure stdlib + already-present pydantic (08-RESEARCH "no Package Legitimacy Audit"). No slopcheck needed. |
</threat_model>
