<objective>
Build the Excel `.xlsx` adapter (ADAPT-03) on the hardened pattern from Plan 01 and the lazy
loader/probe from Plan 02. Double-load the workbook (R3: `data_only=False` formula view +
`data_only=True` data view, `read_only=False`), walk sheets in workbook order row-major, serialize a
deterministic canonical transcript (R3: one line per non-empty extractable cell `Sheet!A1<sep>value`),
apply the locked value->string rule (R3), feed verbatim units to the shared `normalize()`, and route
every value openpyxl cannot faithfully resolve to `unextracted[]` — uncomputed/None-cache formulas,
error cells, formula-evaluates-to-error, charts/images/unreadable features, and parse failures.
Register it as `"excel"`, carry its drops on `Source.extraction`, and conform.

Purpose: deliver ADAPT-03's two faithfulness guarantees — `sheet!cell`-located typed claims, and the
formula-cache gap routed to `unextracted[]` (never fabricated `0`/empty). The faithful rule stays in
exactly one place (`normalize()`); the adapter owns only the honest extract-vs-disclose fork.

Output: `excel_adapter.py`, the registration, and unit tests pinning the per-cell fork. (The
byte-reproducible golden corpus is Plan 04.)
</objective>

  <name>Task 1: The faithful per-cell fork + value->string + transcript serialization (the crux)</name>
  <files>src/newsletters/adapters/excel_adapter.py, tests/test_excel_adapter.py</files>
  <behavior>
    - value_to_str (R3, deterministic, lossless): bool BEFORE int (TRUE/FALSE; bool is an int subclass), int -> str(int) (no ".0"), float -> repr(float) (shortest round-trip; 1.0 -> "1.0", 0.1+0.2 -> exact), datetime/date/time -> .isoformat(), str -> verbatim passthrough. No fixed-precision formatting (no f"{v:.2f}").
    - Per-cell fork (decided from the FORMULA view's data_type, never from the data view): formula cell (data_type=='f') with a None cache -> ('unextracted', reason naming the cell + formula); formula with a cached value -> ('emit', value_to_str(cache)); formula whose cache is itself an error code -> ('unextracted'); error cell (data_type=='e') -> ('unextracted', reason naming the cell + error); blank (formula-view value is None and not a formula) -> ('skip'); literal (str/int/float/bool/datetime) -> ('emit', value_to_str(value)).
    - A formula-with-None-cache cell NEVER yields a claim with text "0" or "" (the ADAPT-03 criterion-2 violation) — it is in unextracted[].
    - Transcript serialization: walking sheets in workbook order, row-major, each emitted cell produces one line "Sheet!A1<sep>value" using the SAME value string as the unit, so every unit is an exact substring of the transcript at its own offset (duplicates get distinct offsets via normalize()'s cursor). A value containing the separator/newline is still emitted verbatim (not escaped) and remains locatable.
    - Merged ranges: the top-left anchor's value is emitted once; the None-valued covered cells are skipped (no phantom blank claims); a 2x2 merge yields exactly 1 claim, not 4, and is NOT recorded as a drop (nothing is lost).
  </behavior>
