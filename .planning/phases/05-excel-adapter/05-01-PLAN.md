<objective>
TASK ZERO — harden the adapter coverage-persistence flaw BEFORE the Excel adapter replicates the
pattern (CONTEXT decision 0; carried from the Phase-4 verifier, RETRO 2026-06-17). Today an
adapter's `unextracted[]` determination lives in adapter instance memory keyed by `source.id`
(`email_adapter.py:134`), so re-`distill()`ing a *persisted* `Source` on a fresh adapter silently
loses those drops and falsely reports `complete=True`. This plan makes adapter coverage
**reconstructable from the `Source` itself** via a TYPED carrier (per R1 — NOT JSON-in-`context`),
retrofits the Email adapter onto it, adds the R2 safety-net, and pins it with a round-trip
coverage-parity test.

Purpose: "No silent drops" must hold ACROSS persistence, not just same-instance — the faithfulness
contract the whole adapter family inherits. The Excel adapter (Plan 03) is built on this hardened
pattern; building it first would re-introduce the flaw.

Output: A new leaf typed carrier (`ExtractionRecord` on `Source.extraction`), a shared codec, a
retrofitted Email adapter with the in-memory dict deleted, the R2 safety-net in `distill()`, and a
parametrized round-trip coverage-parity conformance test. Existing suite stays green.
</objective>
