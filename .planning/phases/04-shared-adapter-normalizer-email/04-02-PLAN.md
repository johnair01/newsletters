## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `.eml` bytes → EmailAdapter | `.eml` files are untrusted/possibly hostile input (arbitrary MIME). The parser must not crash; undecodable / malformed / unreadable content routes to `unextracted[]`, never a fact or an exception that aborts the pipeline. |
| EmailAdapter → distill socket | The adapter returns truth only (`DistillationResult`); it must never publish or construct a published `Review` (HARD RULE, mirrors manual.py:13-17). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-03 | Tampering | mislabeled / hostile charset (`x-unknown-cs`, latin-1 bytes labeled utf-8) | mitigate | Strict fallback ladder catching `(UnicodeDecodeError, LookupError)`; residual U+FFFD detected; never `errors='replace'` silently — routes to U3/U4 (CONTEXT decision 7, RESEARCH Pitfalls 1-2). |
| T-04-04 | Tampering | malformed MIME (missing close boundary, bad headers) | mitigate | Parser surfaces `msg.defects`/`part.defects`; route to U7 rather than raising (RESEARCH `CloseBoundaryNotFoundDefect` verified); still extracts what it can. |
| T-04-05 | Denial of Service | parse/decompression bomb (deeply nested multipart, huge base64 attachment) | mitigate | Adapter never DECODES non-text attachment bytes (records type only → U2); relies on stdlib `iter_attachments`/`walk` not recursing unboundedly; container parts are never `get_payload(decode=True)`'d (Pitfall 6). |
| T-04-06 | Injection (downstream) | HTML body with `<script>`/active content | mitigate | `html.parser` extracts TEXT only and skips `script`/`style`; no HTML is rendered or executed; output is plain text + a lossy disclosure (U5). |
| T-04-07 | Information disclosure (faithfulness) | forwarded `message/rfc822` masquerading as a body part (`is_attachment()==False`) | mitigate | Iterate everything `iter_attachments()` yields and branch on `get_content_maintype()=="message"` → U1; nested mail accounted for, never silently dropped (RESEARCH Pitfall 3, CONTEXT decision 4). |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No package installs (stdlib-only, CONTEXT decision 5); RESEARCH Package Legitimacy Audit is N/A; no legitimacy checkpoint required. A task proposing a new dependency CONTRADICTS a locked decision and must be escalated, not installed. |
</threat_model>
