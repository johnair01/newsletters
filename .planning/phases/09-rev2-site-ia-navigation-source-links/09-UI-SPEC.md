## SITE-05 — Source links: what each cited source links TO (resolved default)

ROADMAP SC4: "The fan-out diagram and **every cited source** render as working links (e.g.
`vision.md` → repo file)." Three link surfaces, one resolution rule:

### The resolved link-target rule (RESOLVES the central ambiguity)

> **Decision:** introduce a single configurable **`source_base_url`** with a default of the project's
> GitHub repo blob root, and a relative fallback. **Default:**
> `https://github.com/nneibaue/newsletters/blob/main/` (matches the Home §7 repo lockup
> `nneibaue / newsletters`, `surfaces.md` L68). A `link_for_source(source, *, base_url)` helper maps
> a cited source to a working URL:
>
> 1. **Doc/file-shaped locators** (a `FreeLocator.text` / `Source.context` that looks like a repo
>    path — e.g. `docs/vision.md`, `CLAUDE.md`, `semantic.py`, `src/newsletters/render.py`):
>    → `{source_base_url}{path}` (e.g. `…/blob/main/docs/vision.md`). The dogfood corpus already
>    carries these as locators (`dogfood.py:218` `locator="docs/"`, `:219` `CLAUDE.md`, `:222`
>    `semantic.py`, `:528` `context="docs/roadmap.md"`).
> 2. **Session/transcript locators** (`SessionLocator`, or a `source_id` like `session-kickoff` /
>    `doc-roadmap` with no file path): → an **in-site anchor** to the surface that records that
>    session (e.g. the Show episode / the Report), NOT a repo file. Resolve via `Site.by_slug` /
>    the source→surface map. Where no in-site target exists, render the evidence chip as **plain
>    text** (current behavior, `render.py:214-217`) rather than a dead link — faithful, never a 404.
> 3. **`base_url=None`** → relative repo paths (`../path`) for fully-offline/self-hosted output.

## Visual-fidelity & build risks

1. **Interactivity → static fidelity gap.** The prototype's Home demo (`who` state) and theme are
   React state; the renderer is static + a tiny toggle. **Risk:** the persona re-cut animation
   (`sg-fade` on `who` change) cannot be reproduced without JS. **Mitigation:** render the
   first/`maintainer` persona's letter as the static default (faithful, no-JS); persona buttons are
   styled labels. The live re-cut is a `web/` phase. Flagged; resolved default = static default.
2. **8-point scale vs the 1:1 token port.** The design system carries editorial half-steps
   (18.5/30/26px). **Risk:** a naive checker flags them. **Mitigation:** the ported `tokens.css`
   values are authoritative for THIS phase (`design-system.md` opening mandate); see Spacing
   Exceptions. The checker must defer to the ported tokens.
3. **`index.html` repurposing breaks existing links.** Today `index.html` IS the library
   (`dogfood.py:677`). Moving it to `library.html` could rot any external bookmark to the old index.
   **Mitigation:** the new `index.html` (Home) links to `library.html` prominently (Home §5 + footer);
   acceptable because the deployed site is pre-1.0 and SITE-02 explicitly makes Home the front door.
4. **Source-link target correctness.** A wrong `source_base_url` yields 404s on every evidence chip.
   **Mitigation:** the rule degrades to **plain text** (never a dead link) when a target cannot be
   resolved (faithful, never suggestive); the GitHub default matches the Home repo lockup; the
   no-dead-link regen test (Wave 3) gates it.
5. **Determinism / byte-stability.** Any `datetime.now()` reaching the rendered HTML breaks SITE-06.
   **Mitigation:** the new Home/board emit nothing time-varying; the regen test (Wave 3) is the guard
   (mirrors the adapters' `EPOCH_ZERO`/deterministic-timestamp discipline used elsewhere in the repo).
6. **CSS-only board.** The "no JS" constraint (SC2, footer claim) rules out any JS-driven column
   filtering. **Mitigation:** pure `display:grid` columns keyed off `Page.gate` at render time; the
   regen test asserts the marker + no `<script>` beyond the existing theme toggle.
