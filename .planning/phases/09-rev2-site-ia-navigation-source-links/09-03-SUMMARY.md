## The source-link model (SITE-05)

`link_for_source(trace, *, base_url=source_base_url, site=None) -> str | None`:

1. **File-path locator** (`_is_file_path_locator`: contains `/` or ends in `.md/.py/.txt/.json/.toml/.cfg/.ini/.yml/.yaml/.rst`, and has no whitespace) → `{base_url}{path}` (the repo blob URL), or `../{path}` when `base_url is None`.
2. **Session/transcript or non-path locator** → an in-site anchor `{slug}.html` when `site.by_slug(trace.source_id)` resolves to a recording surface; otherwise `None`.
3. **Neither** → `None`. The caller (`_ev_chip` / `_fanout_row`) renders **plain text** — never a dead/`href="None"` link.

Default `source_base_url = https://github.com/nneibaue/newsletters/blob/main/` (matches the Home §7 repo lockup `nneibaue / newsletters`, surfaces.md L68). Host-agnostic by design (open-by-default, self-hostable per CLAUDE.md) with the relative fallback for offline export.

**FanoutLink.href population:** `_fanout_row` resolves `link.href` first, then an exact `page.title` match, then `by_slug(slugify(title))`. In the dogfood corpus this links the rows that name real sibling surfaces ("Getting the data models right", "The Weekly Signal — Rev1 is live", "Episode 01 — Building Newsletters in the open") and leaves the descriptive labels ("Designing the semantic spine (peer review)", "Phase 0 — atomic task plan") as plain text — faithful, never a dead link.

**SVG anchors:** `diagrams.fanout(links=...)` wraps each `_box(...)` in `<a href='…'>` using the label→href map (`_fanout_box_links`); the SVG anchor is valid markup and needs no JavaScript. `links=None` preserves the static SVG.
