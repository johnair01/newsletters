<objective>
Build the stdlib TMDL line/indent parser (`_tmdl.py`) — a PURE function that turns one `*.tmdl`
file's text into an ordered list of `(object_path_prefix, verbatim_value)` units, plus a list of
non-extractable signals (e.g. a DirectQuery partition `mode`). This is research-locked L2: a
YAML-like single-tab-indent grammar parsed with the standard library only — NO TMDL parser
dependency, and the parser NEVER evaluates DAX (a measure's expression is a FORMULA, extracted as
text, never a value — the Excel formula-cache crux restated, CONTEXT decision 3 / L2).

Purpose: isolate the one genuinely-new piece of parsing logic (TMDL tokenization) behind a pure,
unit-testable boundary so Wave 2's adapter composes it without re-deriving the grammar. ADAPT-05
success-criterion-1 (extract PBIP/TMDL text) rests on this.

Output: `src/newsletters/adapters/_tmdl.py` (the parser) + `tests/test_tmdl_parser.py` (its RED-first
unit suite). ZERO new dependency (stdlib `re`/`str` only — research Q-B verdict: stdlib is
sufficient; if a TMDL parser dep ever seems essential, STOP and flag — it is not).
</objective>
