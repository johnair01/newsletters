<objective>
Build `PowerBiAdapter` (registered `"powerbi"`) — the front-end that composes the Wave-1 TMDL parser
(`_tmdl`) and PBIR reader (`_pbir`) onto the proven shared spine (`normalize()` / coverage codec /
`deterministic_timestamp`), exactly mirroring `excel_adapter.py`/`pptx_adapter.py` (parse -> units ->
normalize -> carrier -> register -> conform). It delivers ALL of ADAPT-05: PBIP/TMDL+PBIR text ->
content-addressed `Claim(+Trace)` (criterion 1); the row-cap/aggregation `unextracted[]` taxonomy +
the categorical `_R_NO_DATA_ROWS` whole-source disclosure (criterion 2, fail loud); and the `.pbix`
binary deferral (L1 — pbixray DEFERRED, ZERO new dependency).

ENTRYPOINT SHAPE (L4, the planner-finalized contract): a NEW `parse_path(path: str)` walks a PBIP
FOLDER tree (sorted, deterministic) and returns the same `(Source, units, drops)` triple; the existing
byte `parse(raw: bytes, path: str)` handles a single dropped file (`.tmdl`/`.json`/`.bim`) and routes a
`.pbix`/zip-binary byte input to the whole-source `_R_PBIX_BINARY` deferral. Both converge on ONE
serializer. `parse(raw, path)` is also what lets the adapter JOIN the existing parity + determinism
matrices (which call `adapter.parse(raw, path)`).

Purpose: ship the registered, conforming Power BI backend on the shared faithful spine with zero new
dependency and no human checkpoint. Output: `powerbi_adapter.py`, the registration line in
`adapters/__init__.py`, `tests/test_powerbi_adapter.py`, and the adapter joining the parity +
determinism matrices.
</objective>
