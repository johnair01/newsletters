<objective>
Migrate the existing Rev1 dogfood sample sources to content-addressed traces (PROV-01,
success criterion 1; decision D-4 "migration/helper that computes hashes+offsets for existing
sample sources"). Add a helper that, for each Rev1 sample trace, locates its verbatim evidence
span inside the live Source transcript, computes the char offsets, and re-mints the trace via
`Trace.from_source(...)` — so the Rev1 corpus that ships with Newsletters is itself content-
addressed and drift-aware, not just the type system.

Purpose: PROV-01 is only real if the shipped sample corpus demonstrates it. The Rev1 dogfood
content is "Newsletters reporting on building Newsletters" — our first corpus — and it must
practice the trust property it preaches. Migration is in-place and FAITHFUL: it pins existing
spans to offsets, it never rewrites a claim or invents evidence. Anything it cannot locate is
reported, not silently dropped (hard rule: faithful, not suggestive; no silent dropping).

Output: A migration helper in `dogfood.py`, applied in the build path, plus a RED-first
`tests/test_provenance_migration.py` proving the migrated corpus is addressed, self-verifying,
not stale, and that the site build still renders cleanly.
</objective>
