<objective>
Make every Trace content-addressed and source-drift-aware in the typed spine (PROV-01,
success criterion 1). Extend `Trace` with OPTIONAL content-address fields (SHA-256 content
hash of the full Source plus character start/end offsets; the verbatim `span` field already
exists), add a `Source.content_hash()` helper and a self-verifying `Trace.from_source(...)`
constructor, and implement STALE as a COMPUTED property (live source hash != recorded trace
hash) at trace, claim, and distillation granularity — never a stored mutable flag.

Purpose: Positional locators silently mis-attribute when a source is edited. Pinning the
content hash + offsets + verbatim span turns a silent mis-attribution into a deterministic,
visible STALE signal — the trust property PROV-01 exists to guarantee. Per decisions D-1
(SHA-256 of full Source.transcript + char offsets + verbatim span), D-2 (STALE is a computed
property, never a stored flag), and D-4 (fields added optionally for backward-compat).

Output: An extended `Trace`/`Source` in `semantic.py` and a RED-first `tests/test_provenance.py`
proving content-addressing, self-verification, STALE detection, and Rev1 backward-compat.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| edited Source → recorded Trace | A source's content can change after a Trace was minted; the gap between recorded and live content is the trust surface PROV-01 polices. |
| deserialized Trace JSON → spine | A JSON sidecar an operator loads may carry attacker-set content_hash/start/end values. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | edited Source vs recorded Trace.content_hash | mitigate | STALE computed property (Task 3): a content edit deterministically flips dependent claims to STALE instead of silently mis-attributing — the core PROV-01 guarantee. |
| T-03-02 | Spoofing | Trace.span vs offset window | mitigate | Trace.from_source pins span = transcript[start:end] so the stored span is re-checkable against the live source (self-verifying, Task 2). |
| T-03-03 | Information disclosure | content hashing | accept | SHA-256 of transcript is stored, not the transcript; no new PII surface. Hash is a public integrity tag, not a secret. |
| T-03-04 | Tampering | deserialized offsets (end>len, end<start) | mitigate | Trace.from_source validates offset ranges and raises before slicing (Task 2); arbitrary stored offsets are only consumed via self-verification, never to slice a different live source silently. |
| T-03-SC | Tampering | npm/pip/cargo installs | mitigate | None — stdlib `hashlib` + string ops only; no new dependency. No install task; no legitimacy checkpoint required. |
</threat_model>
