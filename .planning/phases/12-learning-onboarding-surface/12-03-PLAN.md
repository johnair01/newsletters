<objective>
Build the faithful learning re-cut logic (L2) and the onboarding-path model (L4) in a NEW cohesive
module `src/newsletters/learning.py`, with the faithfulness + ordering + path tests. This is the
genuinely-new code of the phase; everything else is composition.

The faithfulness crux is enforced here, in code: `learning_surface()` SELECTS, ORDERS, and LINKS
existing traced reviewed claims and traced glossary definitions — it emits only ClaimsBlocks and a
GlossaryBlock (no free ProseBlock, no invented sentence). A glossary term with no traced defining
claim goes to surface.missing[] (the honesty panel shows it), never a fabricated gloss. The
OnboardingPath is a typed ordered list of slug refs — navigation over already-gated surfaces, not a
Surface itself, so it carries no review gate.

Purpose: A teaching surface naturally wants to explain; "faithful, not suggestive" forbids it. The
discipline is encoded as re-arranging existing Claim.text / traced Claim definitions only. The
no-invented-prose test is the load-bearing gate.

Output: learning.py (learning_surface preset + glossary builder + OnboardingPath/OnboardingStep) and
the LEARN-01/02/03 logic tests in tests/test_learning.py.
</objective>
