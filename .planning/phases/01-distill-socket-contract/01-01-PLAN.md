<objective>
Stand up the Phase-1 walking skeleton: the thinnest end-to-end slice that PROVES the distill
socket. After this plan, a caller can register a `ManualBackend` by name, call
`resolve("manual").distill(sources)` through the `DistillPort` boundary, and receive a valid
`DistillationResult` (traced `Distillation` + `Coverage` manifest) that round-trips losslessly
to/from JSON — all with zero AI. This is the backbone Phases 2–12 build on.

The boundary is deliberately **modality-agnostic** (D-01): the same `DistillPort` ->
`Distillation` shape must accommodate all three distill modalities — author-by-hand, generic
file extraction, and agentic interview — so downstream review/render/promote never learns which
one produced a result. Phase 1 builds only the author-by-hand backend (ManualBackend, per D-02);
the extraction and interview backends slot into the identical contract in later phases.

Purpose: Lock the one boundary every distill backend speaks through (SOCK-01), the registry
that selects it (SOCK-02), the by-hand backend that proves it (SOCK-03), and the coverage /
`unextracted[]` contract so no content is silently dropped (SOCK-04). The conformance suite
that *polices* this contract (SOCK-05) lands in Plan 02.

Output: A new top-level leaf module `src/newsletters/locators.py` (the `Locator` union), a new
`src/newsletters/distill/` package (`coverage.py`, `ports.py`, `registry.py`, `manual.py`,
`__init__.py`), a backward-compatible widening of `Trace.locator` in `semantic.py`, the Wave-0
test infrastructure, and `tests/test_distill_socket.py` with the end-to-end + core-contract
tests green.

IMPORT-GRAPH NOTE (load-bearing — corrects the prior cross-AI-flagged circular import): the
`Locator` union lives in a **top-level leaf module** `src/newsletters/locators.py` that imports
NOTHING from `semantic.py` or the `distill` package. `semantic.py` imports the locator types
`from .locators import ...` (exactly mirroring its existing `from .templates import ...` leaf
import), and the `distill` package re-exports them `from ..locators`. This is genuinely acyclic:
`semantic -> locators` (leaf) and `distill -> locators` (leaf), with no path back into `semantic`
triggered during `semantic`'s own initialization. See Task 2's reasoning for why placing the
union *inside* the `distill` package would instead create a cycle.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator-supplied JSON sidecar → `DistillationResult` (on load) | The one untrusted-input surface this phase ships: a serialized result an operator (or a future tool) may load with `model_validate_json`. |
| backend → socket (`distill()` return) | A backend could try to smuggle an untraced/unfaithful claim, or claim full coverage while dropping content. |

> This phase ships **no network, no auth, no PII, no external I/O beyond local JSON sidecars** and
> reading already-loaded `Source` objects. The inputs are Python objects authored in-repo by the
> developer; there is no external untrusted input at this typing/contract layer. The genuine surface
> is narrow: untrusted-JSON deserialization and contract-honesty. We do not invent threats beyond it.
