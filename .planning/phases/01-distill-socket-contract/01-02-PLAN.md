<objective>
Make the contract from Plan 01 *enforceable*: ship a reusable conformance suite (SOCK-05) that
runs ANY backend through the socket and fails it on a contract violation ‚Äî missing traces,
unreported/lying coverage, an unfaithful claim, or any attempt to route around the
no-auto-publish gate. Wire the injectable faithfulness seam into exactly one boundary point so
the rule lives in one place and Phase 3 can swap in deterministic span-containment without
touching any backend.

Purpose: SOCK-05 is the contract's immune system ‚Äî without it, a backend can claim full
coverage while dropping content, or emit an untraced claim, and nothing catches it. This plan
also nails down the load-bearing hard rules as executable assertions (no auto-publish;
AI-optional core).

Output: `src/newsletters/distill/conformance.py` with `assert_conforms()`, a single-place
`_enforce(result, check)` faithfulness hook in `ports.py`, the conformance/hard-rule tests in
`tests/test_distill_socket.py` (a deliberately-broken backend fixture drives the negatives),
and the public barrel updated.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| backend ‚Üí socket (`distill()` return) | The primary boundary this plan polices: any registered backend (manual now; adapters/AI later) crossing into the trusted pipeline. |
| caller ‚Üí socket (attempted publish) | A caller must not be able to reach `Published` through the socket; only the existing review gate may. |

> Same minimal surface as Plan 01 ‚Äî no network/auth/PII/external I/O. This plan's job is to turn
> the contract-honesty boundary into *enforced* assertions rather than ship new attack surface.

| T-01-03 | Tampering | a backend smuggling an untraced/unfaithful claim through the socket | mitigate | `_enforce` (one place, injectable) + `assert_conforms` + `test_conformance_fails_bad_backend` / `test_faithfulness_seam_rejects_untraced` reject it.
