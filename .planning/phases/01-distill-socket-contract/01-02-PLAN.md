<objective>
Make the contract from Plan 01 *enforceable*: ship a reusable conformance suite (SOCK-05) that
runs ANY backend through the socket and fails it on a contract violation — missing traces,
unreported/lying coverage, an unfaithful claim, or any attempt to route around the
no-auto-publish gate. Wire the injectable faithfulness seam into exactly one boundary point so
the rule lives in one place and Phase 3 can swap in deterministic span-containment without
touching any backend.

Purpose: SOCK-05 is the contract's immune system — without it, a backend can claim full
coverage while dropping content, or emit an untraced claim, and nothing catches it. This
RUNTIME suite (not mypy) is the real malformed-backend guard: `@runtime_checkable` `isinstance`
only checks attribute presence, and mypy only ever sees the conforming in-`src` ManualBackend —
so the deliberately-broken backend that PROVES the contract has teeth must be caught at runtime,
here (review MEDIUM-2). This plan also nails down the load-bearing hard rules as executable
assertions (no auto-publish; AI-optional core).

Output: `src/newsletters/distill/conformance.py` with `assert_conforms()`, a single-place
`_enforce(result, check)` faithfulness hook in `ports.py`, the conformance/hard-rule tests in
`tests/test_distill_socket.py` (a deliberately-broken backend fixture drives the negatives),
and the public barrel updated.
</objective>
