<objective>
Add the ONE genuinely new capability of Phase 11: a read-only, no-network, stdlib local-file `Source`
ingest (`capture_files`). It turns selected repo files into content-addressable `Source` records whose
`id` is the POSIX relpath — so the existing renderer resolves each to a working repo link for free, and
hand-authored claims (Wave 2) can content-address (via `Trace.from_source`) to REAL file content. This
is L2 / WORK-01's "read-only adapters at a work codebase, data staying local."

new `worksurface.py` + its tests, disjoint from Plan 11-01 (render.py/fonts), so the two run in parallel
in Wave 1. The end-to-end operator-flow test is scaffolded here (Nyquist Wave-0) and its later stages
are filled by Plans 11-03/04/05.

Purpose: An operator can point Newsletters at a real codebase and capture its files as evidence,
read-only — the literal "install + point read-only adapters, data stays local" of WORK-01.
Output: `capture_files` in a new AI-free module, a read-only/content-addressed test, the AI-isolation
gate extended to `worksurface`, and the e2e scaffold.
</objective>
