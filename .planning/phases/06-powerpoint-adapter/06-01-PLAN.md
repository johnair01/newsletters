<objective>
FRONT FIX (CONTEXT decision 0 / research-locked L1): replace the non-deterministic `now()`
timestamp fallback in ALL adapters with a single shared deterministic sentinel, BEFORE the PPTX
adapter copies the pattern. Today the email adapter spreads `timestamp=` only when a `Date` header
exists (else `Source.timestamp` defaults to `_utcnow()` wall-clock), and the excel adapter branches
on `created is not None` (else the same `_utcnow()` default). For any real document lacking an
intrinsic timestamp, two parses of identical bytes therefore produce NON-equal Sources — breaking
the determinism and round-trip-parity property the whole adapter contract depends on.

This plan introduces `adapters/_timestamps.py` (`EPOCH_ZERO` + `deterministic_timestamp`), retrofits
email + excel to ALWAYS pass `timestamp=deterministic_timestamp(intrinsic)` explicitly (never the
default factory), and adds a determinism test parametrized across email + excel. The PPTX adapter
(06-03) and the pptx determinism param (06-04) join this exact pattern later.

Purpose: lock the shared adapter timestamp pattern to a deterministic sentinel so PPTX inherits a
correct pattern instead of copying the bug. This is the first plan because it changes the shared
adapter convention every later plan builds on.
Output: the `_timestamps.py` helper, two retrofitted adapters, and a cross-adapter determinism test.
</objective>
