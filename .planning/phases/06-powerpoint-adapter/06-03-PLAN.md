<objective>
PPTX ADAPTER (ADAPT-04 / criteria 1+2): build `PptxAdapter` as a structural clone of `ExcelAdapter`.
`parse(raw, path)` lazily loads a single `Presentation` (via 06-02's `_pptx_loader.load_presentation`),
walks `prs.slides` in order and each `slide.shapes` in document order, RECURSING into `GroupShape`
members (L3), and builds one canonical text transcript + ordered verbatim units + an `unextracted[]`.
It emits per-paragraph text from text frames (titles/body/text boxes/auto-shapes), per-cell text from
tables, and speaker-notes paragraphs; it routes every shape the high-level API cannot read — SmartArt
(L2: shape_type is None + diagram `@uri`, with the lxml fallback confirmed in 06-02), charts, pictures,
media, OLE, unreadable group members — to `unextracted[]`. Drops persist on `Source.extraction` via the
shared codec; `distill(sources)` re-derives units from the transcript and routes them through the
shared `normalize()` (the SOLE faithful/hash-minting gate — the adapter NEVER hashes). Timestamp comes
from `deterministic_timestamp(core_properties.created)` (06-01). Locator =
`FreeLocator("Slide N / <shape.name>")`. python-pptx is imported LAZILY only.

Purpose: deliver criteria 1 and 2 — faithful verbatim slide/shape claims and a complete unextracted[]
taxonomy with exact (including nested-group) accounting. Goal-backward: a reviewer must SEE both the
extracted text AND the shapes the extractor skipped.
Output: `pptx_adapter.py`, its registration, and the unit test suite proving the emit/report fork.
</objective>
